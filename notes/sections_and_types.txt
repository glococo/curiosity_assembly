# AVR GNU Assembler: Sections and Data Directives

This document summarizes how the GNU Assembler (GAS) handles memory sections and data allocation for AVR microcontrollers.

## 1. Memory Sections Summary

| Section         | Location | Startup Init?       | Typical Use Case |
|-----------------|----------|---------------------|------------------|
| `.text`         | Flash    | N/A                 | Executable program code. |
| `.rodata`       | Flash    | N/A                 | Read-only constants (accessible via `lpm`). |
| `.progmem.data` | Flash    | N/A                 | Large PROGMEM arrays and strings. |
| `.data`         | SRAM     | **Yes** (Flash copy)| Initialized variables (copied from Flash at boot). |
| `.bss`          | SRAM     | **Yes** (Zeroed)    | Uninitialized global variables (cleared at boot). |
| `.noinit`       | SRAM     | **No**              | Persistent variables that survive a software reset. |
| `.eeprom`       | EEPROM   | N/A                 | Non-volatile configuration and counters. |
| `.fuse`         | Fuses    | N/A                 | Microcontroller hardware configuration. |

---

## 2. Common Data Directives

Directives tell the assembler how to allocate space in the current section.

### Constants and Strings (Flash)
```assembly
.section .rodata
    my_byte:     .byte   0x42             ; Allocate 1 byte
    my_byt2:     .short  0x4232           ; Allocate 2 bytes
    my_word:     .word   0x1234           ; Allocate 2 bytes (Little-Endian: 34 12)
    my_4bytes:   .long   0x42323232       ; Allocate 4 bytes
    status_msg:  .asciz  "System OK"      ; Null-terminated string
    raw_text:    .ascii  "Wait"           ; Non-terminated string
```

### Initialized Variables (SRAM / `.data`)
*Note: In pure assembly, you often use Flash directly. Use `.data` only if the value must be changed at runtime.*
```assembly
.section .data
    runtime_limit: .word 5000             ; Copied from Flash to SRAM at startup
```

### Uninitialized Space (SRAM / `.bss` or `.noinit`)
```assembly
.section .bss
    tx_buffer:   .space  64               ; Reserve 64 zeroed bytes
    temp_var:    .lcomm  local_var, 2     ; Reserve 2 bytes for a local symbol

.section .noinit
    reset_count: .byte   0                ; Will not be zeroed on startup
```

## 3. Best Practices for this HAL

1. **Use `.text.<name>` for Functions**: When defining a function, use a specific subsection to allow the linker to perform garbage collection (`--gc-sections`).
   ```assembly
   .section .text.my_function, "ax", @progbits
   ```
2. **Prefer `.rodata` for strings**: Keep strings in Flash to save precious SRAM. Access them using the `Z` register and `lpm` instruction.
3. **Atomic Toggles**: For modern AVR (Dx/Ex), use the `_OUTTGL` register instead of modifying variables in SRAM whenever possible.
