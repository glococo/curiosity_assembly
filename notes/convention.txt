- The __zero_reg__ is r15
  Never write to r15 unless you know what you are doing

- The "Scratch" Registers (r0, r1)
  R0 and R1 are scratch registers for immediate operations, like MUL.

- Pointer Register Preferences
  AVR has three 16-bit pointer pairs: X (r27:r26), Y (r29:r28), and Z (r31:r30).

  X (r27:r26): Used for SRAM Buffer access, like Ring Buffer
  Y (r29:r28): Used as the Frame Pointer. If you need to create "local variables" on the stack
  Z (r31:r30): Primary use for Program Memory (LPM)

- Call-Saved registers
  All, there are no Call-Used registers
  For sake of security and simplicity, all written registers (beside returns) must be saved

- Arguments passing
  Arguments are passed in even-aligned register pairs, starting from r22 and moving upwards.

   8-bit argument: Use r22.
  16-bit argument: Use r23:r22.
  32-bit argument: Use r25:r24:r23:r22.

  ; Convention: Pass character in r22
    ldi r22, 'A'
    rcall HAL_UART_Transmit

- Macro Naming & Local Labels
  Prevent "Namespace Pollution" and utilize the assembler's local label features.
  Every HAL macro should start with the peripheral name (e.g., HAL_GPIO_, HAL_UART_, HAL_ADC_).
  Local Labels: Always use numeric labels (e.g., 1:, 2:) or the @ symbol inside macros.
  This allows you to use the same macro multiple times in one file without "Duplicate Label" errors.

  1. The "1b / 1f" Numeric Labels (The Best Way)
  The GNU Assembler allows you to use single digits (0–9) as labels. These are not unique.
  You can use 1: in every single function you write, and they will never collide.
 
  To jump to them, you append a b (backward) or an f (forward) to the number.

  2. The .L Prefix (Hidden Descriptive Labels)
  If you want the label to have a name (like error_state:) but you don't want it to show up in
  your Symbol Table or conflict with other files during linking, prefix it with .L.
  The assembler treats any label starting with .L as a temporary local symbol.
  They are invisible to the linker.

  3. Unique Labels in Macros (\@)
  If you put a label inside a macro and call that macro twice in the same file, you'll get
  a "Redefinition of Label" error. To fix this, use the \@ special sequence. The assembler
  replaces \@ with a unique number every time the macro is expanded.
