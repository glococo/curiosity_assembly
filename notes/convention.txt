# AVR Assembly HAL: Convention & Social Contract

To ensure high performance and seamless integration between HAL modules and user code, this framework follows a strict "Social Contract" regarding register usage and naming.

## 1. Register Usage (The Social Contract)

### Specialized Registers
- **`__zero_reg__` (r2)**: Treated as a permanent zero. Used for efficient propagation of carries/borrows (e.g., `adc rd, r2`).
  - **Contract**: Initialized to zero by the HAL startup. **Never** write to `r2`.
- **Scratch Registers (r0, r1)**: Volatile scratch registers.
  - **Contract**: Can be modified by any function (and are modified by `mul`). Do not expect them to persist across calls.

### Pointer Registers
AVR has three 16-bit pointer pairs:
- **X (r27:r26)**: **Callee-saved** Primary pointer for Streaming (printing strings, moving arrays)
- **Y (r29:r28)**: **Callee-saved** Primary pointer for DevAddress, Local variables (stack frame)
- **Z (r31:r30)**: **Callee-saved** Primary pointer for Structures, Ring Buffers, Double Buffers

### Function Arguments & Return Values
Arguments are passed in registers starting from `r25` down to `r22`.
- **8-bit**: Use `r22`.
- **16-bit**: Use `r23:r22`.
- **32-bit**: Use `r25:r22`.
- **Return Values**: Follow the same register mapping as arguments. Sometimes r18 holds success/fail

### Volatile vs. Callee-Saved
- **Volatile**: There are no Volatile registers right now
- **Callee-Saved (r3 - r31)**: The called function **must** preserve all the register it uses, it must PUSH/POP them

## 2. Macro Naming & Local Labels

### Namespace Discipline
Every HAL macro must start with the peripheral name to prevent "Namespace Pollution":
- `HAL_GPIO_`, `HAL_USART_`, `HAL_RTC_`, `MATH_`.

### Local Labels (Avoiding "Duplicate Label" Errors)
1. **Numeric Labels (1f/1b)**: The preferred way. Use single digits (0–9).
   - `1: rjmp 1b` (jump backward to 1).
   - `breq 2f` (jump forward to 2).
2. **Hidden Labels (.L)**: Use `.L` prefix for descriptive labels that shouldn't appear in the Symbol Table.
   - `.L_error_state:`
3. **Unique Macro Labels (\@)**: Inside a macro, use `\@` to generate a unique suffix for every expansion.
   - `loop_\@:`

## 3. Atomic Operations
Whenever possible, utilize the modern AVR hardware features for atomicity:
- Use `_OUTTGL` for toggling pins without Read-Modify-Write.
- Use `_DIRSET` / `_DIRCLR` for configuration.
